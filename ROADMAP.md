# Learning Roadmap

This is a suggested path through the material. Adapt it to your pace.

## Phase 1: Python Basics (Steps 1 to 12)

The goal here is to get comfortable writing Python without looking at examples.

```
Step 1:  Print statements        "Hello, World!" and beyond
Step 2:  Variables               Storing and using data
Step 3:  String concatenation    Combining text together
Step 4:  Math operations         Arithmetic with Python
Step 5:  User input              Making interactive programs
Step 6:  Type conversion         Strings to numbers and back
Step 7:  If statements           Making decisions in code
Step 8:  Loops                   Repeating actions (for, while)
Step 9:  Lists                   Storing collections of items
Step 10: Functions               Reusable blocks of code
Step 11: Mini projects           Calculator, guessing game, todo list
Step 12: Common patterns         Validation, counting, finding max/min
```

**Checkpoint:** You should be able to write a small program (like a calculator or a guessing game) from memory. If you can't, go back and practice more. There's no shame in repetition.

## Phase 2: Data Structures (DSA Steps 1 to 6)

Now you learn the building blocks that every algorithm relies on.

```
Step 1:  Dictionaries            The most important data structure for interviews
Step 2:  Sets                    Fast lookups, removing duplicates
Step 3:  Tuples                  Immutable data, dictionary keys
Step 4:  Advanced lists          Slicing, comprehensions, in-place operations
Step 5:  Strings                 Manipulation, efficient building, palindromes
Step 6:  Collections module      deque, defaultdict, Counter
```

**Checkpoint:** You should know when to use a dictionary vs a set vs a list, and why the choice matters for performance.

## Phase 3: Algorithms (DSA Steps 7 to 15)

This is where you learn to think about problems, not just code them.

```
Step 7:  Time & space complexity  Big O notation, identifying bottlenecks
Step 8:  Two pointers             Sorted arrays, palindromes
Step 9:  Sliding window           Subarray and substring problems
Step 10: Hash map patterns        Frequency counting, prefix sums
Step 11: Sorting & binary search  Divide and conquer
Step 12: Stack & queue            LIFO/FIFO, parentheses matching
Step 13: Recursion & backtracking Subsets, permutations, N-Queens
Step 14: Dynamic programming      Overlapping subproblems, optimization
Step 15: Trees & graphs           DFS, BFS, traversals
```

**Checkpoint:** Given a new problem, you should be able to identify which pattern applies and sketch a solution before writing code.

## Phase 4: Problem Solving (DSA Step 16+)

Apply everything to real problems.

```
Step 16: Problem-solving framework    The 5-step process for any problem
         Easy LeetCode problems       Two Sum, Valid Parentheses, Climbing Stairs
         Medium LeetCode problems     3Sum, Group Anagrams, Coin Change
         Mock interviews              Time yourself, explain out loud
```

**Checkpoint:** You can solve most Easy problems in under 15 minutes and most Medium problems in under 30.

## Daily Practice Routine

Whatever phase you're in, consistency beats intensity.

- **20 minutes minimum:** Type out practice exercises
- **1 to 2 problems:** Solve LeetCode problems at your current level
- **Review:** Look at yesterday's solutions. Can you still write them from memory?

The developers who get good at this aren't the ones who grind 8 hours on a weekend. They're the ones who show up for 30 minutes every day.
