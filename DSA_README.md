# DSA & LeetCode Practice Files

Practice files for mastering Data Structures & Algorithms. Everything you need for LeetCode and coding interviews.

## How to Use

1. **Read the tutorial:** Open `DSA_LEETCODE_TUTORIAL.md` and follow along
2. **Open each practice file** in the `dsa/` folder
3. **Type each exercise 3 to 4 times** until it's muscle memory
4. **Run your code:** `python3 dsa/dsa_stepX_*.py`
5. **Solve LeetCode problems** after mastering each concept
6. **Don't move on** until you can write solutions without looking

## Learning Path

### Week 1-2: Data Structures Mastery
- Master Steps 1-6 (all Python data structures)
- Practice each concept 3-4 times
- Solve 2-3 easy LeetCode problems per day

### Week 3-4: Algorithms Mastery
- Master Steps 7-15 (all algorithms)
- Implement each pattern from scratch
- Solve 3-5 easy/medium problems per day

### Week 5-6: Pattern Recognition
- Focus on Steps 16 (problem-solving framework)
- Identify which pattern to use
- Solve 5-7 problems per day
- Review and optimize solutions

### Week 7+: Interview Prep
- Mock interviews
- Focus on medium problems
- Time yourself (45 min per problem)

## Key Concepts to Master

### Data Structures
- ✅ Dictionaries (hash maps) - O(1) lookups
- ✅ Sets - O(1) membership testing
- ✅ Lists - Slicing, comprehension
- ✅ Collections module - deque, defaultdict, Counter

### Algorithms
- ✅ Two Pointers - Opposite ends, fast/slow
- ✅ Sliding Window - Fixed & variable size
- ✅ Hash Map Patterns - Frequency, prefix sum
- ✅ Binary Search - Template & variations
- ✅ Stack/Queue - LIFO/FIFO patterns
- ✅ Recursion/Backtracking - Generate solutions
- ✅ Dynamic Programming - Overlapping subproblems
- ✅ Tree/Graph Traversal - DFS/BFS

### Problem-Solving
- ✅ 5-step framework
- ✅ Pattern recognition
- ✅ Time/space complexity analysis
- ✅ Edge case handling

## Interview Tips

1. **Clarify:** Ask questions about input/output, constraints
2. **Examples:** Work through examples out loud
3. **Think Aloud:** Explain your thought process
4. **Start Simple:** Begin with brute force, then optimize
5. **Test:** Always test with edge cases
6. **Time Management:** 5 min understand, 10 min think, 20 min code, 5 min test

## Remember

- **Type, don't copy-paste** - Your fingers need to learn
- **Practice 3-4 times** - Repetition builds muscle memory
- **Understand WHY** - Don't just memorize, understand the logic
- **Solve problems** - Apply what you learn immediately
- **Be patient** - Mastery takes consistent practice
